## What changed in the modern platform

FreeSense is not just a renamed firewall image. The current development line introduces a smaller,
explicit package contract; Bootstrap 5 package interfaces; candidate promotion; atomic Suricata
configuration; and CI checks that connect package metadata to the actual build list.

<div class="docs-feature-grid">
  <div><b>Package clarity</b><span>Capabilities, services, support status, resource impact, and direct management links.</span></div>
  <div><b>Security integration</b><span>Suricata 8 events and validation, CrowdSec firewall decisions, and consent-driven ZeroTier routes.</span></div>
  <div><b>Release integrity</b><span>Every artifact records exact source, ports, OS-definition, and FreeBSD revisions.</span></div>
</div>

:::caution[Development status]
FreeSense 1.0 is still under active development on FreeBSD 16-CURRENT. Candidate and Development
images are testing channels, not supported production releases. The download page always reflects
the live release manifest.
:::
